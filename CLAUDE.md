## Project Overview

This is a Hugo static site blog (eallion.com) that uses the "pehtheme" theme and integrates with **Ghost** as headless CMS, plus Mastodon, NeoDB, and various APIs. The site is deployed to both Cloudflare Pages and Alibaba Cloud OSS.

## Key Technologies

- **Hugo**: Static site generator (v0.162.1)
- **TailwindCSS v4**: CSS framework
- **Ghost 6**: Headless CMS (Admin API for content fetch)
- **Node.js**: Build scripts and API integrations
- **pnpm**: Package manager

## Architecture

### Content Flow

1. **Headless CMS**: Ghost 6 manages blog content, fetched via Admin API (requires API key)
2. **LLMs Generation**: `generate-llms.js` produces `LLMs.txt` for AI tool context
3. **Static Generation**: Hugo builds static site from fetched markdown content
4. **CSS Pipeline**: TailwindCSS v4 compiles from `assets/css/input.css` to `assets/css/main.css`
5. **Pagefind**: Build search index after Hugo output
6. **Deployment**: GitHub Actions deploy to Cloudflare Pages + Alibaba Cloud OSS

### Directory Structure

- `content/blog/`: Hugo markdown files (generated by Ghost fetch scripts)
- `assets/css/`: TailwindCSS source and compiled styles
- `assets/data/`: JSON data fetched from external APIs
  - `friends/`: Friend links data
  - `penta/`: Page view statistics
  - `stats/`: Aggregated stats (Mastodon, Steam, NeoDB, GitHub)
- `scripts/`: Node.js build scripts and API integrations
  - `ghost-fetch-posts.js`: Fetch posts from Ghost Admin API
  - `ghost-fetch-pages.js`: Fetch pages from Ghost Admin API
  - `fetch-stats.js`: Multi-API stats aggregator (5 services)
  - `generate-llms.js`: Generate LLMs.txt for AI tool context
- `layouts/`: Hugo template overrides
  - `_default/mastodon.html`: 嘀咕 Mastodon timeline page
  - `_default/stats.html`: Site stats page
  - `partials/comments.html`: Mastodon-powered blog comments
- `static/`: Static assets served as-is
- `config/_default/hugo.toml`: Main Hugo configuration
- `config/_default/params.toml`: Site parameters (comments, social, etc.)

### External Integrations

- **Ghost 6** (ghost.eallion.com): Headless CMS via Admin API
- **Mastodon** (e5n.cc): Social media posts for "嘀咕" page, blog comments
- **NeoDB**: Book and movie tracking data
- **Cloudflare Pages**: Primary hosting

- **Penta** (penta.eallion.com): Page view statistics
- **Steam**: Gaming stats
- **GitHub API**: Repository stats

## Ghost API Details

- **Must use Admin API** (not Content API) — all imported posts have `visibility: paid`, Content API won't return HTML/body
- **Pagination**: Admin API uses `?page=N&limit=50`, response has `meta.pagination`
- **Filter**: `?filter=status:published` for published content
- **Formats**: `?formats=html,lexical` — fetch both HTML and lexical JSON
- **Lexical parsing**: `lexical` field is a flat char-indexed JSON string; reconstruct by sorting numeric keys and joining values. Markdown cards have `{ root: { children: [{ type: 'markdown', version: 1, markdown: "..." }] } }`
- **Turndown fallback**: When lexical has no Markdown card, convert HTML to markdown via turndown with GFM plugin
- **PUT requires `updated_at`** — optimistic locking field, must be sent with updates
- **Admin API is read+write** — API key is sensitive, fetch scripts only do GET reads

## Environment Setup

1. Copy `.env.example` to `.env.local`
2. Configure required API tokens and endpoints:
   - `GHOST_ADMIN_API_KEY`: Ghost Admin API key (read+write, keep secret)
   - `NEODB_ACCESS_TOKEN`: NeoDB API access
   - `STEAM_WEB_API_KEY`: Steam API
   - `GITHUB_TOKEN`: Optional — 60 req/h without, 5000 req/h with

## Build Process

1. **Clean**: Remove old blog posts and Hugo output
2. **Ghost Fetch**: Pull posts and pages from Ghost Admin API, generate markdown with TOML frontmatter
3. **Stats Fetch**: Aggregate data from Mastodon, Steam, NeoDB, Penta, GitHub (per-API failure logging with `✗` prefix)
4. **CSS Build**: TailwindCSS v4 processes `assets/css/input.css`
5. **LLMs Generation**: `generate-llms.js` produces `LLMs.txt`
6. **Hugo Build**: Static site generation with minification
7. **Pagefind**: Build search index

- `layouts/`: Template overrides

## Important Notes

- Content is fetched from Ghost Admin API at build time, not stored locally
- All imported Ghost posts have `visibility: paid` — Content API will NOT return body
- Mastodon blog comments: `params.toml` `[comment].enable` toggle (global) + per-article frontmatter check in `single.html`
- Stats page uses 4 data layers: Hugo computed, external APIs (5 services), friends JSON, client-side JS
- All API failures in stats fetch fall back to defaults with visible per-API `✗` error logging
- Site supports Chinese content with CJK language configuration
- GitHub Actions deploy to both Cloudflare Pages and Alibaba Cloud OSS
